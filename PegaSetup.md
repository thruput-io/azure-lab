# Pega Integration Guide

This guide describes how to connect a **Pega Platform** application to:
- **Azure Event Hubs** (Kafka endpoint, via Application Gateway) for message produce/consume
- **Apicurio Schema Registry** (Confluent-compatible API, via Application Gateway) for Avro schema support

Both properties files are self-contained and delivered via **Actions → Download client.properties → Run workflow**.

---

## Prerequisites

- **Pega Platform 8.7 or later** (Kafka Data Set support)
- Both properties files from **Actions → Download client.properties → Run workflow**:
  - `kafka-client.properties` — Kafka broker connectivity (SASL/PLAIN)
  - `schema-client.properties` — Kafka broker + Schema Registry + OAuth (self-contained Avro client config)

> **Important:** Do not edit these files. They are generated by Terraform and contain the correct
> credentials for this environment. If anything is wrong, the fix is in the infrastructure — not in the file.

---

## Platform constraint: Kafka broker authentication

Azure Event Hubs port 9093 supports **only SASL/PLAIN with `$ConnectionString`** for Kafka clients.
`SASL/OAUTHBEARER` is **not supported** on the Kafka endpoint (tested and confirmed, see `deviations.md` DEV-001).
OAuth is only available via AMQP (port 5671), which Confluent KafkaSerdes does not use.

This is the documented Azure Event Hubs + Confluent integration pattern — not a workaround.

## Step 1 — Retrieve the properties files

1. Go to **Actions → Download client.properties → Run workflow** in the GitHub repository.
2. Download the `client-properties` artifact.
3. Extract both files:
   - `kafka-client.properties`
   - `schema-client.properties`

### `kafka-client.properties` — Kafka broker only

```properties
bootstrap.servers=<CUSTOM_DOMAIN_NAME>:9093
security.protocol=SASL_SSL
sasl.mechanism=PLAIN
sasl.jaas.config=org.apache.kafka.common.security.plain.PlainLoginModule required \
  username="$ConnectionString" \
  password="<EVENT_HUB_CONNECTION_STRING>";

topic=orders.placed
checks_topic=checks.kafka
```

### `schema-client.properties` — Kafka broker + Schema Registry + OAuth (Avro clients)

```properties
bootstrap.servers=<CUSTOM_DOMAIN_NAME>:9093
security.protocol=SASL_SSL
sasl.mechanism=PLAIN
sasl.jaas.config=org.apache.kafka.common.security.plain.PlainLoginModule required \
  username="$ConnectionString" \
  password="<EVENT_HUB_CONNECTION_STRING>";

schema.registry.url=https://<CUSTOM_DOMAIN_NAME>/apis/ccompat/v7

bearer.auth.issuer.endpoint.url=https://login.microsoftonline.com/<TENANT_ID>/oauth2/v2.0/token
bearer.auth.client.id=<CLIENT_ID>
bearer.auth.client.secret=<CLIENT_SECRET>
bearer.auth.scope=api://<APICURIO_CLIENT_ID>/.default
```

> The `bearer.auth.*` properties are consumed natively by the Confluent Schema Registry client
> (`kafka-avro-serializer` / `kafka-avro-deserializer`). No custom token handling is needed.

## Step 2 — Configure Kafka in Pega

### 2a — Kafka without schema (plain JSON/text)

Use `kafka-client.properties`.

In **Pega Designer Studio**:

1. Navigate to **Records → Integration-Resources → Kafka Configuration**.
2. Create a rule (e.g. `EventHubKafkaConfig`).
3. Set **Bootstrap servers** to `<CUSTOM_DOMAIN_NAME>:9093`.
4. Under **Additional client properties**, paste the contents of `kafka-client.properties`.

Key properties Pega needs:

```properties
security.protocol=SASL_SSL
sasl.mechanism=PLAIN
sasl.jaas.config=org.apache.kafka.common.security.plain.PlainLoginModule required \
  username="$ConnectionString" \
  password="<EVENT_HUB_CONNECTION_STRING>";
```

### 2b — Kafka with Avro schema (Schema Registry)

Use `schema-client.properties`.

In addition to the Kafka Configuration rule above, configure the Confluent Schema Registry client:

```properties
schema.registry.url=https://<CUSTOM_DOMAIN_NAME>/apis/ccompat/v7
bearer.auth.issuer.endpoint.url=https://login.microsoftonline.com/<TENANT_ID>/oauth2/v2.0/token
bearer.auth.client.id=<CLIENT_ID>
bearer.auth.client.secret=<CLIENT_SECRET>
bearer.auth.scope=api://<APICURIO_CLIENT_ID>/.default
```

These properties are passed directly to `kafka-avro-serializer` / `kafka-avro-deserializer`.
The SR client acquires the OAuth token natively — no custom token handling is needed in Pega.

---

## Step 3 — Create a Kafka Data Set Rule

1. Navigate to **Records → Integration-Resources → Data Set**.
2. Create a **Kafka** data set (e.g. `OrderPlacedDataSet`).
3. Set **Kafka Configuration** to `EventHubKafkaConfig`.
4. Set **Topic** to `orders.placed`.
5. Set **Message format** to match your Pega model:
   - **JSON/Text** — use `kafka-client.properties`
   - **Avro** — use `schema-client.properties` (Schema Registry required)

---

## Step 4 — Verify connectivity

Produce a test message and verify Pega can consume it.

Example payload (Avro `OrderPlaced` schema):

```json
{"order_id": "pega-test-001", "product": "turnaround", "quantity": 1, "timestamp": 1713648000000}
```

Verify in Pega:
- The record appears in Data Set browse/run.
- Fields map correctly to your target data class.
- No Kafka consumer or authentication errors in logs.

> **Tip:** If no records appear, check consumer offset settings (`latest` vs `earliest`).
> For first-time testing, use `earliest` to consume messages already in the topic.

## Compatibility Summary

| Item | Status | Notes |
|---|---|---|
| Kafka broker auth (`SASL/PLAIN`) | ✅ Supported | Only supported auth on AEH port 9093 |
| Kafka broker auth (`SASL/OAUTHBEARER`) | ❌ Not supported | AEH Kafka endpoint does not support it (DEV-001) |
| Bootstrap via App Gateway `:9093` | ✅ Supported | SNI must match `CUSTOM_DOMAIN_NAME` |
| Schema Registry (Apicurio, Confluent-compat) | ✅ Supported | `/apis/ccompat/v7` endpoint |
| Schema Registry OAuth (`bearer.auth.*`) | ✅ Supported | Entra ID client credentials flow |
| `kafka-client.properties` | ✅ Self-contained | Broker only — JSON/text use case |
| `schema-client.properties` | ✅ Self-contained | Broker + SR + OAuth — Avro use case |

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `SASL authentication failed` | Wrong `$ConnectionString` | Re-run "Download client.properties" to get fresh credentials |
| `User is not authenticated` (SR) | `bearer.auth.*` misconfigured | Verify `bearer.auth.scope` matches `api://<APICURIO_CLIENT_ID>/.default` |
| `Schema not found` | Schema not registered | Schema is auto-registered on first produce by `kafka-avro-serializer` |
| No messages consumed | Offset at `latest`, topic empty | Switch to `earliest` for initial test |
| TLS/SNI error | Wrong bootstrap server | Must match `CUSTOM_DOMAIN_NAME` exactly |
